# 附录 A：常用端口号表

本附录用于快速查询企业网络中常见协议、端口号、传输层协议和排错关注点。学习端口号时不要只背数字，要同时理解三个问题：

- 这个端口对应什么业务。
- 客户端和服务器哪一侧监听端口。
- 防火墙、安全组、ACL 或 NAT 需要放行哪个方向的流量。

端口号不是安全策略本身。真实生产环境中，安全策略还应结合源地址、目的地址、用户身份、应用识别、时间段、日志审计和最小权限原则。

## A.1 端口号基础

TCP 和 UDP 都使用端口号区分不同应用。IP 地址用于找到主机，端口号用于找到主机上的具体服务。

例如，用户访问内部 Web 系统：

```text
客户端 10.28.10.25:51532 -> 服务器 10.28.60.20:443
```

这里 `10.28.60.20` 是服务器地址，`443` 是 HTTPS 服务端口，`51532` 是客户端临时端口。防火墙通常需要允许办公网访问服务器的 TCP 443，而不是固定允许客户端源端口 51532。

常见端口范围如下。

| 范围 | 名称 | 说明 |
| ---: | --- | --- |
| `0-1023` | Well-known ports | 常见基础服务端口，例如 DNS、HTTP、HTTPS、SSH |
| `1024-49151` | Registered ports | 许多数据库、中间件和应用服务端口 |
| `49152-65535` | Dynamic/private ports | 客户端临时端口常见范围，不同操作系统可能不同 |

排错时要注意：服务端监听的是固定端口，客户端源端口通常是临时端口。抓包和会话表中看到高位端口并不一定代表异常。

## A.2 基础网络服务端口

| 服务 | 端口 | 协议 | 常见方向 | 用途 | 排错关注点 |
| --- | ---: | --- | --- | --- | --- |
| FTP Data | `20` | TCP | 服务器到客户端或协商生成 | FTP 主动模式数据通道 | 主动/被动模式、防火墙 ALG、临时端口 |
| FTP Control | `21` | TCP | 客户端到服务器 | FTP 控制连接 | 能登录不代表能传文件 |
| SSH | `22` | TCP | 客户端到服务器/设备 | 加密远程登录、SCP、SFTP | 设备是否启用 SSH、账号权限、ACL |
| Telnet | `23` | TCP | 客户端到服务器/设备 | 明文远程登录 | 生产环境不建议开放 |
| SMTP | `25` | TCP | 邮件服务器之间 | 邮件投递 | 出口常被运营商或安全策略限制 |
| DNS | `53` | UDP/TCP | 客户端到 DNS 服务器 | 域名解析 | UDP 常规查询，TCP 用于大响应/区域传送 |
| DHCP Server | `67` | UDP | 客户端到服务器 | DHCP 请求目的端口 | 跨网段需要 DHCP Relay |
| DHCP Client | `68` | UDP | 服务器到客户端 | DHCP 响应目的端口 | 终端拿到 `169.254.x.x` 时重点检查 |
| TFTP | `69` | UDP | 客户端到服务器 | 简单文件传输、设备升级 | 无认证，生产环境慎用 |
| HTTP | `80` | TCP | 客户端到服务器 | Web 明文访问 | 可能跳转到 HTTPS |
| POP3 | `110` | TCP | 客户端到邮件服务器 | 邮件收取 | 老旧邮件客户端场景 |
| NTP | `123` | UDP | 客户端到时间服务器 | 时间同步 | 日志、证书、认证依赖准确时间 |
| IMAP | `143` | TCP | 客户端到邮件服务器 | 邮件同步 | 通常被 IMAPS 替代 |
| SNMP | `161` | UDP | 监控系统到设备 | 轮询设备状态 | Community/用户权限、源地址限制 |
| SNMP Trap | `162` | UDP | 设备到监控系统 | 主动告警上报 | 方向与 SNMP 轮询相反 |
| BGP | `179` | TCP | 路由器之间 | 边界网关协议邻居 | 源地址、TTL、ACL、TCP 建连 |
| LDAP | `389` | TCP/UDP | 客户端到目录服务器 | 目录查询和认证 | 域控、应用认证常见依赖 |
| HTTPS | `443` | TCP | 客户端到服务器 | 加密 Web 访问 | 证书、SNI、代理、TLS 版本 |
| SMB/CIFS | `445` | TCP | 客户端到文件服务器 | Windows 文件共享、域访问 | 跨网段开放风险高 |
| Syslog | `514` | UDP/TCP | 设备到日志服务器 | 日志集中收集 | UDP 可能丢日志，TCP 更可靠 |
| LDAPS | `636` | TCP | 客户端到目录服务器 | 加密 LDAP | 证书链和域名匹配 |
| SMTPS | `465` | TCP | 客户端到邮件服务器 | 加密 SMTP 提交 | 与 `587` 使用方式不同 |
| SMTP Submission | `587` | TCP | 客户端到邮件服务器 | 邮件客户端提交 | 常配合 STARTTLS 和认证 |
| IMAPS | `993` | TCP | 客户端到邮件服务器 | 加密 IMAP | 企业邮件客户端常见 |
| POP3S | `995` | TCP | 客户端到邮件服务器 | 加密 POP3 | 老旧邮件收取场景 |

## A.3 认证、目录和运维管理端口

| 服务 | 端口 | 协议 | 用途 | 企业场景 |
| --- | ---: | --- | --- | --- |
| TACACS+ | `49` | TCP | 网络设备 AAA | 设备登录认证、命令授权、审计 |
| Kerberos | `88` | TCP/UDP | 域认证 | Windows AD 登录、票据认证 |
| DNS | `53` | UDP/TCP | 域名解析 | AD、Web、邮件、应用访问都依赖 |
| NTP | `123` | UDP | 时间同步 | 域认证、日志审计、证书校验 |
| LDAP | `389` | TCP/UDP | 目录查询 | 应用对接 AD/LDAP |
| LDAPS | `636` | TCP | 加密目录查询 | 应用认证的推荐方式之一 |
| RADIUS Auth | `1812` | UDP | 认证授权 | 802.1X、无线认证、VPN 登录 |
| RADIUS Accounting | `1813` | UDP | 计费/审计 | 记录用户上线、下线和流量信息 |
| Syslog TLS | `6514` | TCP | 加密日志传输 | 安全要求较高的日志集中收集 |

AAA 相关端口常见于交换机、无线控制器、防火墙、VPN 网关和堡垒机之间。排查登录失败时，不要只看账号密码，还要检查设备到认证服务器的路由、源接口、防火墙策略、共享密钥和时间同步。

## A.4 Windows 和办公网络常见端口

| 服务 | 端口 | 协议 | 用途 | 风险提示 |
| --- | ---: | --- | --- | --- |
| RPC Endpoint Mapper | `135` | TCP | Windows RPC 端点映射 | 不建议向不可信网络开放 |
| NetBIOS Name | `137` | UDP | NetBIOS 名称服务 | 老旧 Windows 网络 |
| NetBIOS Datagram | `138` | UDP | NetBIOS 数据报 | 老旧 Windows 网络 |
| NetBIOS Session | `139` | TCP | NetBIOS 会话 | 常被 SMB 445 替代 |
| SMB/CIFS | `445` | TCP | 文件共享、域访问 | 横向移动风险高，应严格分区 |
| RDP | `3389` | TCP/UDP | Windows 远程桌面 | 必须限制源地址并配合堡垒机/MFA |
| WinRM HTTP | `5985` | TCP | Windows 远程管理 | 管理网内使用 |
| WinRM HTTPS | `5986` | TCP | 加密 Windows 远程管理 | 证书和权限要配套 |

办公网到服务器区的 SMB、RDP、WinRM 不应大范围放开。推荐做法是只允许运维网、堡垒机或指定管理终端访问指定服务器。

## A.5 数据库和中间件端口

| 服务 | 默认端口 | 协议 | 用途 | 常见安全要求 |
| --- | ---: | --- | --- | --- |
| Microsoft SQL Server | `1433` | TCP | SQL Server 数据库 | 只允许应用服务器访问 |
| Oracle Listener | `1521` | TCP | Oracle 数据库监听 | 限制源地址，避免办公网直连 |
| MySQL/MariaDB | `3306` | TCP | MySQL 数据库 | 禁止公网暴露 |
| PostgreSQL | `5432` | TCP | PostgreSQL 数据库 | 使用强认证和最小访问范围 |
| Redis | `6379` | TCP | 缓存/键值数据库 | 默认不应跨安全区开放 |
| MongoDB | `27017` | TCP | 文档数据库 | 禁止匿名和公网访问 |
| Elasticsearch | `9200` | TCP | REST API | 需要认证和访问控制 |
| Elasticsearch Transport | `9300` | TCP | 集群节点通信 | 只允许集群节点之间访问 |
| Kafka | `9092` | TCP | 消息队列 | 注意 advertised listener 和防火墙 |
| RabbitMQ AMQP | `5672` | TCP | 消息队列 | 只允许应用组件访问 |
| RabbitMQ Management | `15672` | TCP | Web 管理界面 | 管理口应放在管理区 |

数据库端口一般不应向普通办公网、访客网或公网开放。典型路径应是：

```text
用户 -> Web/API 服务器 -> 数据库
```

而不是：

```text
用户 -> 数据库
```

## A.6 VPN、隧道和路由协议相关端口

有些网络协议不使用 TCP/UDP 端口，而是直接使用 IP 协议号。防火墙策略里要区分“端口号”和“协议号”。

| 协议/服务 | 端口或协议号 | 类型 | 用途 | 说明 |
| --- | --- | --- | --- | --- |
| ISAKMP/IKE | UDP `500` | UDP 端口 | IPsec 协商 | 建立安全关联 |
| IPsec NAT-T | UDP `4500` | UDP 端口 | NAT 环境下 IPsec | 穿越 NAT 常用 |
| ESP | IP 协议号 `50` | IP 协议 | IPsec 加密数据 | 不是 TCP/UDP 端口 |
| AH | IP 协议号 `51` | IP 协议 | IPsec 认证头 | 企业中相对少见 |
| L2TP | UDP `1701` | UDP 端口 | L2TP 隧道 | 常与 IPsec 组合 |
| PPTP | TCP `1723` | TCP 端口 | 老旧 VPN | 还依赖 GRE，不建议新建 |
| GRE | IP 协议号 `47` | IP 协议 | 隧道封装 | 不是 TCP/UDP 端口 |
| RIP | UDP `520` | UDP 端口 | 动态路由 | 小型或实验网络 |
| OSPF | IP 协议号 `89` | IP 协议 | 动态路由 | 不使用 TCP/UDP 端口 |
| BGP | TCP `179` | TCP 端口 | 动态路由 | 路由器之间建立 TCP 会话 |

如果 IPsec VPN 第一阶段失败，重点检查 UDP 500、双方公网地址、预共享密钥、IKE 参数和 NAT。若协商成功但业务不通，继续检查 UDP 4500/ESP、加密域、路由、NAT 豁免和安全策略。

## A.7 Web 和代理常见端口

| 服务 | 端口 | 协议 | 用途 |
| --- | ---: | --- | --- |
| HTTP | `80` | TCP | 明文 Web |
| HTTPS | `443` | TCP | 加密 Web |
| HTTP Alternate | `8080` | TCP | 测试 Web、代理或应用服务 |
| HTTPS Alternate | `8443` | TCP | 管理界面或备用 HTTPS |
| Squid Proxy | `3128` | TCP | 常见代理服务 |
| SOCKS Proxy | `1080` | TCP | SOCKS 代理 |

Web 访问异常时，需要区分是 DNS 失败、TCP 连接失败、TLS 握手失败、HTTP 状态码异常，还是应用登录异常。端口连通只是其中一步。

## A.8 网络设备和监控常见端口

| 服务 | 端口 | 协议 | 常见设备 | 排错关注点 |
| --- | ---: | --- | --- | --- |
| SSH | `22` | TCP | 交换机、路由器、防火墙、服务器 | VTY/用户/ACL/密钥算法 |
| HTTPS 管理 | `443` | TCP | 防火墙、无线控制器、管理平台 | 证书、管理地址、安全策略 |
| SNMP | `161` | UDP | 网络设备、服务器、UPS | Community 或 SNMPv3 用户 |
| SNMP Trap | `162` | UDP | 网络设备到监控平台 | Trap 目的地址和源接口 |
| Syslog | `514` | UDP/TCP | 网络设备到日志平台 | 日志级别、时间同步 |
| NetFlow/sFlow/IPFIX | 常见 `2055`、`6343`、`4739` | UDP/TCP | 设备到流量分析平台 | 采样率、源地址、收集器端口 |

管理类端口应放在管理 VLAN 或管理区，不应从全网开放。运维人员访问设备建议经过堡垒机、VPN 或零信任访问平台，并保留登录审计。

## A.9 安全策略放行示例

假设企业使用以下网段：

| 区域 | 网段 | 说明 |
| --- | --- | --- |
| 办公网 | `10.28.10.0/24` | 员工终端 |
| 服务器区 | `10.28.60.0/24` | 内部业务服务器 |
| 管理区 | `10.28.250.0/24` | 运维终端、堡垒机、监控 |
| 访客网 | `10.28.80.0/24` | 访客无线 |

推荐策略示例：

| 源区域 | 目的区域/地址 | 服务 | 动作 | 说明 |
| --- | --- | --- | --- | --- |
| 办公网 | `10.28.60.20` | TCP `443` | 允许 | 员工访问内部 HTTPS 系统 |
| 办公网 | DNS 服务器 | UDP/TCP `53` | 允许 | 域名解析 |
| 办公网 | NTP 服务器 | UDP `123` | 允许 | 时间同步 |
| 管理区 | 网络设备管理地址 | TCP `22` | 允许 | SSH 管理设备 |
| 管理区 | 服务器区 | TCP `3389` | 按需允许 | 建议只允许堡垒机 |
| 访客网 | Internet | TCP `80/443`、UDP/TCP `53` | 允许 | 访客上网 |
| 访客网 | 内网 RFC1918 地址 | Any | 拒绝 | 隔离访客访问内网 |

不要写成“办公网到服务器区 Any 允许”。初期看似省事，后续会导致安全边界失效。

## A.10 排错速查

| 现象 | 优先检查端口/协议 | 下一步 |
| --- | --- | --- |
| 不能打开网页 | TCP `80/443`、DNS `53` | 先 `nslookup`，再测试 TCP 端口 |
| 能 ping 服务器但业务打不开 | 业务端口，例如 `443`、`3306` | 检查服务监听、防火墙策略和服务器本机防火墙 |
| 终端拿不到地址 | UDP `67/68` | 检查 VLAN、Trunk、DHCP Relay、地址池 |
| 设备无法被监控 | UDP `161`、Trap `162` | 检查 SNMP 配置、源地址、ACL |
| 日志平台收不到日志 | UDP/TCP `514` 或 TCP `6514` | 检查设备日志源地址、路由和日志级别 |
| 域账号登录慢或失败 | DNS `53`、Kerberos `88`、LDAP/LDAPS `389/636`、NTP `123` | 检查 DNS 指向、时间偏差、域控连通性 |
| IPsec VPN 协商失败 | UDP `500/4500`、ESP `50` | 检查 NAT、IKE 参数、预共享密钥、安全策略 |
| BGP 邻居不起来 | TCP `179` | 检查邻居地址、源接口、路由、ACL |
